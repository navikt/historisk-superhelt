# AGENTS.md ‚Äî navikt/historisk-superhelt

Case management system for one-time payments on simple health benefits (orthoses, wigs, footwear, prostheses, travel expenses). Replaces InfoTrygd routines HT-MV, SB-SA and GE-PP.

## Architecture

```
root (Maven multi-module)
‚îú‚îÄ‚îÄ backend/    Spring Boot 4 + Kotlin ‚Äì REST API + Kafka + JPA
‚îú‚îÄ‚îÄ frontend/   React 19 + TanStack Router/Query + Nav Aksel DS
‚îú‚îÄ‚îÄ libs/       Internal Kotlin libraries (common-types, pdl, helved, tilgangsmaskin, dokarkiv, oppgave, pdfgen)
‚îú‚îÄ‚îÄ mocks/      Mock server (Kotlin) + mock-oidc for local development
‚îú‚îÄ‚îÄ pdfgen/     Letter generator (Docker)
‚îî‚îÄ‚îÄ e2e/        Playwright tests (run against live app)
```

**Case flow:**
1. Case handler creates `Sak` ‚Üí fills in information ‚Üí sends for attestation
2. Attestant attests ‚Üí case is finalised ‚Üí `Vedtak` saved
3. On approval: `Utbetaling` sent to Helved via Kafka (`historisk.utbetaling.v1`), decision journalised in Dokarkiv and distributed
4. Helved returns statuses on `helved.status.v1`

**Auth:** Azure AD via Nais Wonderwall (sidecar). Locally: mock-oidc + Wonderwall in Docker. Texas is used for token exchange to other services.

## Code Standards

### Backend (Kotlin)
- Domain layer: data classes at package root
- `db/` ‚Äî JPA entities + JPA repositories
- `rest/` ‚Äî REST controllers + DTO types
- `*Repository.kt` ‚Äî app repository (access control, mapping to domain)
- `*Service.kt` ‚Äî business logic; `@Transactional` goes here, not on repositories
- Repository classes always return domain model (never JPA entity) to callers
- Unit tests use AssertJ (`assertThat`) with JUnit 5
- Integration tests use Testcontainers (Postgres); requires Docker
- Test data factories in `*TestData.kt` objects (e.g. `SakTestData`, `UtbetalingTestData`)
- Mock external dependencies with mockito-kotlin

### Database / Flyway
- Always add a Flyway migration for schema changes ‚Äî naming: `V{n}__{description}.sql`
- **Never modify existing migration files**

### Access control
- `@PreAuthorize("hasAuthority('READ')")` / `'WRITE'` on repository methods
- `@tilgangsmaskin.harTilgang(#fnr)` for person-based access
- Never bypass auth checks

### Kafka
- Producer: `UtbetalingKafkaProducer` sends `UtbetalingMelding` to Helved
- Consumer: `UtbetalingStatusConsumer` receives statuses and updates `UtbetalingStatus`
- Status machine: `UTKAST ‚Üí KLAR_TIL_UTBETALING ‚Üí SENDT ‚Üí MOTTATT ‚Üí BEHANDLET ‚Üí UTBETALT | FEILET`

### Frontend (TypeScript / React)
- Types are auto-generated from the backend OpenAPI spec via `npm run openapi-ts` ‚Üí `generated/`
- Import generated types via `@generated` alias (e.g. `import type { Sak } from "@generated"`)
- Derive local utility types from generated types: `type SakStatusType = Sak['status']`
- TanStack Router with file-based routing in `src/routes/`
- TanStack Query for all data fetching (generated by openapi-ts)
- Shared components in `src/common/`
- **Always** use Nav Aksel Design System (`@navikt/ds-react`) ‚Äî no custom CSS or Tailwind
- Spacing via `Box paddingBlock/paddingInline` with `space-*` tokens ‚Äî never `p-*`/`m-*`
- Responsive design with `{ xs: ..., md: ..., lg: ... }` props
- When running Biome to validate changes: run only on changed files (`npx biome check <file>`), not the whole project
